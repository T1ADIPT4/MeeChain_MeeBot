# MeeBot Patch: Validation Checklist

ถัดไปที่ผมแนะนำ:

- Trigger workflow บน GitHub: Actions → Apply Patch & Validate → Run workflow
- ตั้งค่า `patch_file = meebot-fixes.patch`

ตรวจสอบผลลัพธ์หลัง workflow รันเสร็จ:

1. PR ถูกสร้างบน branch `meebot/fix-jest-deps` (หรือ reuse PR เดิม)
   - PR title: 🧩 MeeBot Patch: Fix jest config & Admin props
   - PR body: This PR was auto-generated by MeeBot CI.
2. Labels: `patch`, `auto-generated`, `needs-review`
3. Assignees / Reviewers: `t1adipt4`, `meebot-reviewer` (ถ้ามี)
4. `patch-status.json` ถูก commit เข้า `gh-pages/docs/status/patch-status.json`
5. Viewer: `https://<owner>.github.io/<repo>/docs/` สามารถโหลด `docs/status/patch-status.json` และแสดงผลสถานะ

Local fallback (PowerShell) — ใช้เมื่อต้องการผลลัพธ์ทันทีและมี Git บนเครื่อง

```powershell
# เปลี่ยนไปที่ repository root
Set-Location -Path "C:\Path\To\Repo"

# สร้าง branch ใหม่และแอปพลายแพตช์
.\packaging\apply_patch_and_push.ps1 -PatchFile "meebot-fixes.patch" -Branch "meebot/fix-jest-deps"
```

หมายเหตุ: สคริปต์จะตรวจสอบว่า `git` อยู่ใน PATH และจะหยุดถ้าไม่พบ
